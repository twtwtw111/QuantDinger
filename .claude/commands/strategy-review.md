# /strategy-review — 策略代码审查

## 用途
检查策略代码是否符合 QuantDinger ScriptStrategy 规范，发现逻辑漏洞和潜在风险。

## 使用方式
```
/strategy-review
然后粘贴策略代码
```

## 检查清单

### 1. 格式规范检查
- [ ] 包含 `on_init(ctx)` 函数
- [ ] 包含 `on_bar(ctx, bar)` 函数
- [ ] 函数签名正确（不多不少参数）

### 2. 沙箱安全检查
- [ ] 未使用禁止模块（requests/httpx/os/subprocess/open）
- [ ] 未使用 `eval()` 或 `exec()`
- [ ] 未访问文件系统
- [ ] 所有 import 都在允许列表内

### 3. 逻辑合理性检查
- [ ] 买卖信号逻辑完整（有买就有卖）
- [ ] 有止损机制
- [ ] 无空指针风险（`if indicator is None: return`）
- [ ] 无过度频繁交易风险（有冷却期或频率限制）
- [ ] 无前瞻性偏差（不使用未来数据）

### 4. 风险控制检查
- [ ] 仓位大小有限制
- [ ] 有最大持仓时间或退出条件
- [ ] 止损比例合理（建议 2-5%）
- [ ] 止盈比例合理（盈亏比 > 1.5）
